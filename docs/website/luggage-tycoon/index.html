Luggage Tycoon

Thom Castermans
Willem Sonke

We created a program to simulate a system of conveyor belts that transport luggage items.

The system could potentially be used as a tool for modelling for example a luggage system at an airport. However, we have decided to make our application more like a game, in which the player has to transport luggage from one place to another.

The main focus in our program lies on user interaction and good visualisation. Hence, the program should be easy to use, it should not be hard to get to know the program and it should look good.

The game

Click on the images below to get a larger version.

This is what a luggage simulation looks like. Luggage has to be transported from the enter blocks to the leave blocks on the opposite sides of the room.

The game has several levels. In this level the player has to build a quite complicated conveyor belt to get the luggage to the right place.

We have created a menu system, in which the player can customize the game (for example the language can be set to Dutch or English and the mouse sensitivity can be changed).

Problems and solutions

Of course, to create a luggage simulation that looks realistic, the luggage should behave in a physically accurate way. For example, if one piece of luggage drops onto another one, they should form a stack. We have implemented physics using the JBullet physics library.

The player can click on the 3D view to place and edit conveyor belts. To find out where the user clicked in the 3D space, we implemented a variant of Bresenham's algorithm.
